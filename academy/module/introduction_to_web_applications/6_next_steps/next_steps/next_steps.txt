Следующие шаги

	-В этом модуле мы изучили некоторые, но далеко не все основы веб-приложений.
	Теперь у нас должно быть фундаментальное понимание того, как строится
	веб-приложение, как оно работает и какие опасности оно может представлять
	для корпоративной среды.

	Важно использовать практический подход для дальнейшего развития нашего
	понимания и применения тем, изучаемых в этом модуле. Мы рекомендовали
	рассматривать материал в сочетании с разработкой небольшого веб-приложения.
	Некоторые следующие шаги, которые можно предпринять:
		
		
		1. 	Настройка виртуальной машины с веб-сервером

		2. 	Создать HTMLстраница

		3. 	Создайте его с CSS

		4. 	Добавьте несколько простых функций с помощью JavaScript

		5. 	Запрограммируйте простое веб-приложение

		6. 	Подключите ваше веб-приложение к базе данных

		7. 	Экспериментируйте с API

		8. 	Протестируйте свое приложение на наличие различных уязвимостей и дыр
			в безопасности

		9. 	Попробуйте настроить свой код и конфигурации, чтобы закрыть
			уязвимости

	Разработка небольшого веб-приложения обеспечит гораздо более глубокое
	понимание структуры и функциональности. Изучение того, как настроить и
	управлять таким веб-сервером, роль базы данных и то, как отдельные фрагменты
	кода связаны друг с другом, является бесценным опытом.
