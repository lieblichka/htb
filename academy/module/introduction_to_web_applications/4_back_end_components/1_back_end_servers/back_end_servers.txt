*** Back End Servers
Внутренние серверы 

	Внутренний сервер — это аппаратное обеспечение и операционная система, на
	которой размещены все приложения, необходимые для запуска веб-приложения.
	Это настоящая система, выполняющая все процессы и выполняющая все задачи, из
	которых состоит все веб-приложение. Внутренний сервер будет соответствовать
	уровню доступа к данным . 
		
		https://en.wikipedia.org/wiki/Data_access_layer
	
	### Software 
	Программного обеспечения 
		
		Внутренний сервер содержит 3 других внутренних компонента:
			
			-Web Server
			-Database
			-Development Framework
			
			~~~~___ LINK ___~~~ img/backend-server.jpg
		
		Другие программные компоненты на внутреннем сервере могут включать
		гипервизоры , контейнеры и WAF.
			
			https://en.wikipedia.org/wiki/Hypervisor

		Существует множество популярных комбинаций «стеков» для внутренних
		серверов, которые содержат определенный набор внутренних компонентов.
		Вот некоторые распространенные примеры:
			
			LAMP 	Linux, Apache, MySQL, and PHP.
				
				https://en.wikipedia.org/wiki/LAMP_(software_bundle)

			WAMP 	Windows, Apache, MySQL, and PHP.
				
				https://en.wikipedia.org/wiki/LAMP_(software_bundle)#WAMP


			WINS 	Windows, IIS, .NET, and SQL Server
				
				https://en.wikipedia.org/wiki/Solution_stack

			MAMP 	macOS, Apache, MySQL, and PHP.
				
				https://en.wikipedia.org/wiki/MAMP

			XAMPP 	Cross-Platform, Apache, MySQL, and PHP/PERL.
				
				https://en.wikipedia.org/wiki/XAMPP

		Мы можем найти исчерпывающий список стеков веб-решений в этой статье .
			
			 https://en.wikipedia.org/wiki/Solution_stack

	### Hardware
	Аппаратное обеспечение 
		
		Внутренний сервер содержит все необходимое оборудование. Возможности
		мощности и производительности этого оборудования определяют, насколько
		стабильным и отзывчивым будет веб-приложение. Как ранее обсуждалось в
		ArchitectureВ разделе многие архитектуры, особенно для крупных
		веб-приложений, предназначены для распределения их нагрузки по множеству
		внутренних серверов, которые совместно работают для выполнения одних и
		тех же задач и доставки веб-приложения конечному пользователю.
		Веб-приложения не обязательно должны работать непосредственно на одном
		внутреннем сервере, но могут использовать центры обработки данных и
		службы облачного хостинга, которые предоставляют виртуальные хосты для
		веб-приложения.
